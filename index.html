Access model

Typical User Roles & Permissions

A modern security operations platform needs clear role boundaries so each user sees the right tools on web or mobile.

Admin / Owner

  • Access: Full platform administration across all sites, teams, and settings.
  • Controls: User provisioning, billing, site setup, audit logs, and compliance oversight.

Operations Manager / Supervisor

  • Access: Scheduling, live monitoring, incident approval, and roster publishing.
  • Controls: Shift edits, guard coordination, escalations, and response follow-up.

Client / Site Manager

  • Access: View-only reports and site summaries for their assigned locations.
  • Controls: SLA visibility, incident outcomes, and post-order review without operational edits.

Security Guard / Officer

  • Access: Mobile app only for clock in/out, patrol scanning, incident reporting, and roster lookup.
  • Controls: Tour completion, panic alerts, evidence capture, and personal shift visibility.

Dispatcher

  • Access: Real-time monitoring, live map awareness, and emergency response coordination.
  • Controls: Active incident routing, radio/chat escalation, and dispatch history tracking.
2025–2026 direction

Modern Technology Stack Suggestion

The recommended direction stays mobile-first and cloud-based, with real-time communication and secure access for field teams.

Architecture & Frontend

  • Architecture: Mobile-first, cloud-based deployment instead of on-premise infrastructure.
  • Mobile app: Flutter or React Native for one shared iOS and Android codebase.
  • Web dashboard: React / Next.js or Vue.js for operations, reporting, and client portals.

Backend & Data

  • Backend: Node.js + TypeScript, Python with FastAPI, or C# /.NET.
  • API: REST endpoints with WebSockets for live tracking, messaging, and status changes.
  • Database: PostgreSQL as the core relational store with Redis for caching.
  • Time-series: TimescaleDB extension or InfluxDB for guard location history.

Realtime, Maps & Security

  • Realtime: Firebase, Supabase Realtime, or Pusher for live movement and response updates.
  • Maps: Mapbox GL JS or Google Maps Platform with places support.
  • Authentication: JWT with refresh tokens and multi-factor authentication using TOTP.
  • Security: Biometric mobile login, row-level security, and encryption at rest.

Mobile App Features Checklist

Core field operations

  • Offline support with sync when connectivity returns.
  • Background location optimized for battery efficiency.
  • Push notifications through FCM / APNs.

Checkpoint & evidence tools

  • QR and NFC reader support for patrol checkpoints.
  • Camera capture and file upload for incident evidence.
  • Clock in / out workflow with site-aware validation.

Safety & authentication

  • Panic button with silent alert and live location sharing.
  • Biometric login using Face ID or fingerprint on supported devices.
  • Secure role-based access aligned to officer, dispatcher, and supervisor workflows.
